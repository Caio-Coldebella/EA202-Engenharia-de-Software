Atividades Práticas 6

Integração ao MongoDB e Componentes Vue

Na aula passada, vimos como podemos usar o Express para criar um servidor de backend baseado em JavaScript. Entretanto, como vocês devem ter percebido, ao tentar fazer o exercício que passamos, trabalhar com arquivos é bem complicado. Dessa forma, esse servidor de backend não funcionará sozinho. Ele precisará de serviços de retaguarda, como por exemplo, um banco de dados. Vamos utilizar o banco de dados MongoDB para suportar um importante serviço de retaguarda: persistência da informação. MongoDB é um banco de dados não relacional que armazena objetos JSON. O MongoDB é implementado na forma de um processo servidor (mongod) que normalmente fica rodando em background na máquina em que está instalado. Entretanto, ele não precisa estar instalado na mesma máquina em que o servidor de backend e em nosso caso, estaremos fazendo isso distribuindo nossa aplicação em múltiplos containers Docker. Se vamos fazer isso usando mais de um container docker, precisaremos criar uma pequena rede interna, para poder acessar esses serviços de retaguarda. Para isso, precisaremos criar uma rede interna docker. Podemos fazer isso usando o seguinte comando docker:

docker network create ea202

É necessário criar essa sub-rede interna, para que os diferentes containers possam se enxergar mutuamente, e funcionar sem maiores problemas. Cada container, quando for criado, deve mencionar o acesso a essa rede, caso acesse serviços de retaguarda da rede. 

Instalando o MongoDB

O MongoDB pode ser instalado normalmente em uma máquina. Em nosso caso, entretanto, vamos usar a mesma alternativa que utilizamos no caso do node.js que é instalar o MongoDB a partir de sua imagem Docker. Com isso, a instalação fica tão simples como criar o arquivo mongo.sh com o seguinte script bash:

#! /bin/bash
docker run \
-it \
--name mongo \
--rm \
-d \
-h mongo \
--net=ea202 \
mongo:latest

Ao ser executado, esse script carrega o mongo em um container docker e o disponibiliza na rede ea202 criada anteriormente para nossa aplicação. Observe que para que nossa aplicação funcione, será necessário também modificar nosso script node.sh. Para facilitar, disponibilizamos aqui também uma nova versão do script já com o comando novo, que deve ser copiado no arquivo node.sh e ter o flag executável setado com chmod ugo+x node.sh :

#! /bin/bash
SCRIPTPATH="$( cd "$(dirname "$0")" ; pwd -P )"
docker run \
-it \
--name node.js \
--rm \
-d \
-h node.js \
-w /node \
-v ${SCRIPTPATH}/:/node/ \
-p 3000:3000 \
--net=ea202 \
node

Para interagir com o MongoDB a partir do Node.js é necessário uma interface de programação Javascript para esse banco de dados. Na nossa aplicação vamos utilizar o Mongoose como interface para o MongoDB.

MongoDB: Estrutura

Acesso ao MongoDB via Mongoose

MongoDB é acessível via rede. Para acesso na mesma máquina utiliza-se a URL mongodb://mongo:27017/ acrescida da base de dados (database) da aplicação, por exemplo:

mongodb://mongo:27017/alunosDB

Utilizando o Mongoose:

var mongoose = require("mongoose");
mongoose.connect("mongodb://mongo:27017/alunosDB");

Temos a variável mongoose "conectada" ao MongoDB para operar a base de dados alunosDB.

Mongoose: Schemas

Para armazenar e recuperar uma informação em um banco de dados é necessário antes definir a estrutura dessa informação (meta-informação).

No caso do MongoDB essa estrutura é denominada Schema e consiste na definição dos atributos e respectivos tipos do objeto JSON a ser manipulado.  Exemplo:

var software = {
"nome": String,
"descricao": String,
"versao": String,
"release": Date,
"preco": Number,
"plataformas": Array,
"licenca": ObjectId
};

var Schema = mongoose.Schema;
var softwareSchema = new Schema(software);

Mongoose: Modelos e Coleções

Modelos são objetos da aplicação que espelham os objetos armazenados no banco de dados (documentos). Todos os documentos são armazenados, alterados e recuperados do banco de dados por meio desses modelos. Modelos são especificados por meio de Schemas e Coleções.

Para criar um modelo no Mongoose:

var swModel = mongoose.model("software", softwareSchema);

O primeiro argumento da função model (software) especifica o nome de uma coleção (collection), ou seja, um conjunto de objetos JSON com a mesma estrutura, estrutura essa passada no segundo argumento da função model (Schema). 

Manipulando a variável swModel manipulamos a respectiva coleção na base de dados.

Mongoose: Queries

Objetos representando modelos (como swModel) possuem métodos para manipular a base de dados (queries), por exemplo:
create: cria um novo documento na base de dados;
find: recupera documentos da base de dados;
update: atualiza documentos na base de dados;
remove: remove documentos da base de dados
findOne, findOneAndUpdate, findOneAndRemove: apenas um documento é operado - o primeiro a ser encontrado.

Esses métodos recebem um objeto JSON com os atributos que expecificarão os documentos a serem operados no banco de dados; e uma função que recebe dois parâmetros: um código de sucesso/falha e os documentos recuperados do banco de dados (se for o caso).

Queries podem ter seu escopo limitado por meio da construção where. Exemplo:

swModel.find().where('preco').lt(1000).gt(2000);

Ao invés de realizar queries sobre a classe do modelo (swModel) podemos instanciar um modelo para executar um query específico, por exemplo:

soft = new swModel();
soft.nome = "Matlab";
soft.descricao = "software para computacao numerica";
...
soft.plataformas = ["Windows", "Linux", "MacOS"];
soft.licenca = {"_id": "56d45f787f56d32a4357"};

Agora podemos manipular esse objeto no banco de dado via as mesmas funções de query (soft.save(), soft.find(), etc.) mas sem a necessidade de passar um objeto JSON nessas funções (apenas a função de erro), por exemplo:

soft.save(function(erro) { ... });

Mongoose: Arquivos Schema/Modelo

É prática comum especificar o Schema e seu modelo associado em um arquivo específico. Na aplicação exemplo esse arquivo encontra-se no diretório models (/models/mongo.js):

var mongoose = require("mongoose");
conn1 = mongoose.createConnection('mongodb://mongo:27017/alunosDB',      {useNewUrlParser: true});
var Schema = mongoose.Schema;
var alunoSchema = new Schema({
    "ra": String,
    "nome": String,
    "curso": String
});
module.exports = conn1.model('alunos', alunoSchema);

No servidor app.js:

var mongoOp = require('./models/mongo');  // modelo mongoOp

MongoDB Shell

É possível utilizar o MongoDB via um shell de comandos. Essa forma é útil para depuração e testes.

O shell é implementado no programa mongo. Os principais comandos são:

show dbs : lista os databases;
use <dbs> : usa um database específico;
db.dropDatabase() : remove todo o database em uso;
db.create() : cria uma nova coleção;
show collections : lista as coleções do database em uso
db.<collection>.find() : acessa os objetos presentes na coleção;
db.<collection>.insertOne() : insere um objeto JSON na coleção;
db.<collection>.updateOne() : atualiza um objeto JSON na coleção;
db.<collection>.deleteOne() : remove um objeto JSON na coleção;
db.<collection>.drop() : remove toda a coleção.

Veja https://docs.mongodb.com/manual/mongo/ para outras operações via shell.

Testando a Integração com o MongoDB

Abra um terminal no Ubuntu. Crie um diretório para armazenar os arquivos da aula de hoje e dê um mv para lá. Crie os arquivos node.sh e mongo.sh como indicado nos tópicos anteriores, tornando-os arquivos executáveis com chmod ugo+x. 

Baixe os arquivos de exemplo e descompacte o mesmo em seu diretório local. 

Crie então uma aplicação nova, por exemplo:

> express AppMongo

Mude o diretório para AppMongo e faça: 

> mkdir models
> cp ../Exemplos/mongo.js ./models
> cp ../Exemplos/index.js ./routes/index.js
> cp ../Exemplos/index.html  index.html

Instale o mongoose:

> npm install mongoose

Inspecione as funções de middleware acrescentadas no routes/index.js e observe as operações no banco de dados.

Altere o servidor e o banco de dados para incorporar os novos parâmetros do aluno introduzidos na atividade anterior.
 
Gere a aplicação: npm install
Execute o servido: npm start
Abra o browser e acesse o endereço http://localhost:3000/

Execute testes agora com a interface. Tente introduzir dados semelhantes ao que você fez na aula passada, para diferentes RAs, e recuperar depois. Pare o servidor com um CTRL-C e reinicie o mesmo com um novo npm start e você verá que os dados estão de fato sendo armazenados no banco de dados MongoDB. 

Componentes Vue

Uma boa fonte de conhecimentos sobre o vue é o Vue Guide. Na aula de hoje, estudaremos a seção do Vue Guide que fala sobre Components.

Tipicamente, um component no vue, possui a seguinte estrutura:

// Define a new component called todo-item
Vue.component('todo-item', {
  template: '<li>This is a todo</li>'
})

var app = new Vue(...)

Uma vez que o componente foi definido, ele pode ser utilizado no HTML:

<ol>
  <!-- Create an instance of the todo-item component -->
  <todo-item></todo-item>
</ol>

Em princípio, o template pode conter qualquer tipo de código HTML. Veja uma introdução sobre componentes no Vue Guide - Components.

Ferramentas de Programação e Teste

Existem diversas ferramenta que podem ser utilizadas para implementação de aplicações web e, entre as ferramentas online, pode-se utilizar o js bin ou o liveweave.  Ambos possuem visualização com atualização automática da página criada conforme o usuário trabalha.

Já entre as ferramentas instaláveis, lembre-se que os próprios navegadores são interpretadores de html/javascript e podem ser usados para a vizualização do resultado de um código feito no editor de texto sua preferência, bastando estar com a extensão esperada.

No entanto uma IDE pode vir ajudar bastante. Assim, o uso do VSCode em conjunto com o plugin LiveServer te fornece, entre outras coisas, intellisense (autocompletar expressões, funcionalidade de encontrar declaração original etc) e uma visualização dinâmica diretamente no chrome, sem precisar ficar atualizando a página. As vantagem principal desse método é o intellisense mas pode-se ainda utilizar recursos do computador sem a necessidade de buscar-los na internet.

Deliveries

Persistência em um Container MongoDB

No exemplo de integração com o MongoDB, como o container roda dentro de si o próprio Banco de Dados, uma vez que o container mongo é destruído, os dados se perdem. Isso pode ser modificado, utilizando-se a chave -v na chamada do docker e passando um diretório local, que funcionará como diretório persistente onde os dados podem ser armazenados. Dessa forma, quando o container for destruído, os dados serão preservados, e quando o container for chamado novamente, os dados armazenados nesse diretório serão utilizados pelo container. 

Como primeira atividade da aula de hoje, descubra como criar um container MongoDB que permita ter seus dados armazenados externamente, por meio de uma chave -v passada na criação do container. 

Desenvolva um novo script mongo.sh que chame esse container, agora utilizando um diretório externo como fonte de armazenamento.
